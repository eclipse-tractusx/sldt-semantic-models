Aspect Model Manufactured Parts Quality Information Aspect Model

urn:samm:io.catenax.manufactured_parts_quality_information:3.0.0#ManufacturedPartsQualityInformation
For Catena-X(CX) business case Quality Management. Supported technical features: delta update feature, meta information feature and additionalInformationList key-value pairs. The supported model features are described in detail in the Catena-X Quality KIT. Manufactured parts quality information data model is used to exchange information how a list of parts was manufactured with focus on information that might help to better understand the root cause for the quality task.

    Description
    This regular expressions enforces a given date according ISO 8601 local date. General format is YYYY-MM-DD. YYYY can be 0000..9999. MM-DD is checked together: months 01,03,05,07,08,10,12 must have DD in the range of 01..31 days, months 04,06,09,11 must have DD in the range of 01..30 days, month 02 must have DD in the range of 01..29 days.

Record status

The record operation enumeration can be used to realize delta update concept. An example using scenario that data delivery runs weekly: Week0: All data is transferred -> all rows have recordStatus=new. Week1: Only the changed data is transferred: That means new rows, or updated or deleted rows: New rows get recordStatus=new, changed rows get recordStatus=changed and deleted/removed rows get recordStatus=deleted.
Name

Characteristic
